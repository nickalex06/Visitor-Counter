# Руководство по использованию Visitor Counter

## 1. Назначение проекта

Visitor Counter — это прототип автономного счётчика проходов на базе STM32F401CCU6 Black Pill, ToF-датчика расстояния VL53L0X и microSD-карты.

Устройство фиксирует пересечение контролируемой зоны, увеличивает счётчик и сохраняет результаты на microSD-карту.

## 2. Что нужно для работы

### Аппаратная часть

- STM32F401CCU6 Black Pill
- VL53L0X ToF-датчик расстояния
- microSD-модуль
- microSD-карта
- USB-C кабель
- Powerbank или USB-питание
- Провода Dupont

### Программная часть

- STM32CubeIDE
- STM32CubeProgrammer
- Git, если проект скачивается через `git clone`

## 3. Как скачать проект

Можно скачать проект через GitHub:

```bash
git clone https://github.com/nickalex06/Visitor-Counter.git
```

Или скачать архив вручную:

```text
GitHub → Code → Download ZIP
```

После скачивания архив нужно распаковать.

## 4. Как открыть проект в STM32CubeIDE

Открой STM32CubeIDE и выбери:

```text
File → Import → Existing Projects into Workspace
```

Дальше в поле `Select root directory` выбери папку:

```text
Visitor-Counter/firmware/Visitor_Counter
```

После этого проект должен появиться в STM32CubeIDE.

## 5. Как собрать проект

В STM32CubeIDE выбери:

```text
Project → Build Project
```

Если сборка прошла успешно, в папке `Debug/` появится файл прошивки `.elf`.

По умолчанию он находится примерно здесь:

```text
firmware/Visitor_Counter/Debug/Visitor_Counter.elf
```

Название файла может отличаться, если проект был переименован.

## 6. Как прошить STM32 через USB-C

Плата STM32F401CCU6 Black Pill может прошиваться через USB DFU-режим.

Порядок действий:

1. Отключить STM32 от компьютера.
2. Зажать кнопку `BOOT0`.
3. Подключить STM32 к компьютеру через USB-C.
4. Если плата не определяется, нажать и отпустить `NRST`, продолжая удерживать `BOOT0`.
5. Отпустить `BOOT0`.
6. Открыть STM32CubeProgrammer.
7. Справа выбрать режим `USB`.
8. Нажать `Connect`.
9. Перейти в раздел `Erasing & Programming`.
10. В `File path` выбрать `.elf` файл из папки `Debug/`.
11. Включить `Verify programming`.
12. Включить `Run after programming`.
13. Нажать `Start Programming`.

После прошивки программа запустится на STM32.

## 7. Подключение модулей

### VL53L0X

| VL53L0X | STM32 |
|---|---|
| VIN | 3V3 |
| GND | GND |
| SCL | PB6 |
| SDA | PB7 |
| GPIO1 | не подключается |
| XSHUT | не подключается |

### microSD-модуль

| microSD-модуль | STM32 |
|---|---|
| VCC | 5V |
| GND | GND |
| CS | PA4 |
| SCK | PA5 |
| MISO | PA6 |
| MOSI | PA7 |

Важно: в данном прототипе использовался microSD-модуль с пином `VCC`, которому для нормальной работы потребовалось питание от `5V`. Если используется другой модуль с пином `3V3`, его нельзя подключать к 5V.

## 8. Как запустить счётчик

1. Вставить microSD-карту в microSD-модуль.
2. Подключить STM32 к powerbank или USB-питанию.
3. Устройство включится, но счётчик ещё не начнёт работу.
4. Нажать и отпустить кнопку `KEY`.
5. После отпускания `KEY` начинается новая сессия подсчёта.

Кнопки:

| Кнопка | Назначение |
|---|---|
| KEY | начать сессию подсчёта |
| NRST | перезапустить STM32 |
| BOOT0 | войти в режим прошивки через USB DFU |

## 9. Индикация LED

Синий LED используется как индикатор состояния зоны перед датчиком.

| Поведение LED | Значение |
|---|---|
| LED не горит | зона свободна, состояние `EMPTY` |
| LED горит | объект перекрыл зону, состояние `BLOCKED` |
| LED мигает сериями | ошибка инициализации |

Если после нажатия `KEY` LED мигает сериями, значит произошла ошибка.

Примерная расшифровка:

| Количество миганий | Ошибка |
|---|---|
| 1 | не удалось смонтировать SD-карту |
| 2 | не удалось инициализировать VL53L0X |
| 3 | не удалось создать стартовые файлы |
| 4 | не удалось записать `COUNT.TXT` |

## 10. Выходные файлы на microSD

После работы устройства на microSD-карте появляются файлы:

| Файл | Назначение |
|---|---|
| `COUNT.TXT` | текущее итоговое значение счётчика |
| `EVENTS.TXT` | события `BLOCKED` и `PASS` |
| `LOG.TXT` | общий лог расстояний, событий и состояний |

### Пример `COUNT.TXT`

```text
count=6
```

### Пример `EVENTS.TXT`

```text
--- START ---
time_ms=12522, event=BLOCKED, distance=171, count=0
time_ms=14188, event=PASS, distance=1228, count=1
```

### Пример `LOG.TXT`

```text
--- START ---
time_ms=11515, distance=NO_TARGET, count=0, state=EMPTY
time_ms=12522, event=BLOCKED, distance=171, count=0
time_ms=12522, distance=171, count=0, state=BLOCKED
time_ms=14188, event=PASS, distance=1228, count=1
```

## 11. Принцип работы алгоритма

Датчик VL53L0X измеряет расстояние до стены или свободного пространства напротив датчика.

В коде задаётся константа:

```c
#define WALL_DISTANCE_MM 1200
```

Она соответствует расстоянию до противоположной стены или рабочей границе контролируемой зоны.

Если человек проходит между датчиком и стеной, измеренное расстояние становится существенно меньше. Тогда система переходит в состояние `BLOCKED`.

Когда человек выходит из зоны, расстояние возвращается к стене или датчик выдаёт `NO_TARGET`. Тогда система возвращается в состояние `EMPTY`, увеличивает счётчик и записывает событие `PASS`.

Используются два порога:

```c
#define BLOCK_DELTA_MM 200
#define CLEAR_DELTA_MM 80
```

Это нужно, чтобы учесть погрешность датчика и избежать ложных срабатываний из-за колебаний на 1–3 см.

## 12. Как изменить расстояние до стены

Если устройство устанавливается в другом месте, нужно изменить значение:

```c
#define WALL_DISTANCE_MM 1200
```

Например, если расстояние до стены 800 мм:

```c
#define WALL_DISTANCE_MM 800
```

После изменения нужно заново собрать проект и прошить STM32.

## 13. Ограничения прототипа

Текущая версия использует один датчик расстояния.

Поэтому устройство считает факт пересечения зоны, но не определяет направление движения. Оно не различает вход и выход.


Также качество работы зависит от:

- положения датчика;
- расстояния до стены;
- скорости прохода;
- отражающих свойств одежды и объектов;
- стабильности питания;
- корректной установки microSD-карты.

## 14. Типичный сценарий использования

1. Установить датчик напротив прохода.
2. Проверить, что расстояние до противоположной стены соответствует `WALL_DISTANCE_MM`.
3. Вставить microSD-карту.
4. Подключить устройство к powerbank.
5. Нажать и отпустить `KEY`.
6. Провести тестовые проходы.
7. Отключить питание.
8. Вынуть microSD-карту.
9. Открыть `COUNT.TXT`, `EVENTS.TXT`, `LOG.TXT` на компьютере.
