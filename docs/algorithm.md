# Алгоритм подсчёта проходов

Устройство использует ToF-датчик VL53L0X для измерения расстояния до объекта перед датчиком. Датчик устанавливается напротив стены или другой стабильной поверхности. Расстояние до этой поверхности задаётся константой `WALL_DISTANCE_MM`.

## Основная идея

Если проход свободен, датчик видит стену или получает состояние `NO_TARGET`.

Если человек проходит между датчиком и стеной, измеренное расстояние становится заметно меньше расстояния до стены. Это означает, что контролируемая зона занята.

Алгоритм имеет два состояния:

```text
EMPTY    — зона свободна
BLOCKED  — перед датчиком находится объект
```

Переход из `EMPTY` в `BLOCKED` происходит, когда расстояние становится меньше порога обнаружения.

Переход из `BLOCKED` в `EMPTY` происходит, когда расстояние снова становится близким к расстоянию до стены или датчик получает `NO_TARGET`.

Один проход засчитывается именно при переходе:

```text
BLOCKED → EMPTY
```

## Пороговые значения

Для настройки используются следующие константы:

```c
#define WALL_DISTANCE_MM 1200
#define SENSOR_TOLERANCE_MM 30

#define BLOCK_DELTA_MM 200
#define CLEAR_DELTA_MM 80

#define BLOCK_THRESHOLD_MM (WALL_DISTANCE_MM - BLOCK_DELTA_MM)
#define CLEAR_THRESHOLD_MM (WALL_DISTANCE_MM - CLEAR_DELTA_MM)
```

`WALL_DISTANCE_MM` — расстояние от датчика до противоположной стены.

`SENSOR_TOLERANCE_MM` — ожидаемая погрешность измерения. В текущей версии она учитывается через запас между порогами.

`BLOCK_DELTA_MM` — насколько расстояние должно стать меньше расстояния до стены, чтобы объект считался обнаруженным.

`CLEAR_DELTA_MM` — насколько расстояние должно вернуться к расстоянию до стены, чтобы зона считалась свободной.

## Гистерезис

В алгоритме используются два разных порога:

```text
BLOCK_THRESHOLD_MM — порог входа в состояние BLOCKED
CLEAR_THRESHOLD_MM — порог выхода обратно в EMPTY
```

Это уменьшает ложные срабатывания, которые могли бы появиться из-за шума измерений или погрешности датчика.

## Обработка NO_TARGET

Если VL53L0X не получает валидное расстояние, это состояние записывается как `NO_TARGET`.

В текущей логике `NO_TARGET` трактуется как свободная зона. Это удобно для установки в проходе, где противоположная стена может быть дальше рабочей дальности датчика.

## Запись данных

На microSD-карту записываются три файла:

```text
COUNT.TXT   — текущее значение счётчика
EVENTS.TXT  — события BLOCKED и PASS
LOG.TXT     — периодический лог расстояния, состояния и счётчика
```

После каждого события `PASS` значение счётчика обновляется в `COUNT.TXT`.

## Ограничение текущей версии

Один датчик позволяет считать факт пересечения зоны, но не позволяет надёжно определить направление движения. Для различения входа и выхода требуется минимум два датчика, расположенных последовательно вдоль направления движения.
