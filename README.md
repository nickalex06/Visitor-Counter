# Visitor Counter

**Visitor Counter** — прототип автономного счётчика проходов на базе микроконтроллера STM32F401CCU6, ToF-датчика расстояния VL53L0X и записи данных на microSD-карту.

Устройство фиксирует прохождение человека через контролируемую зону и сохраняет результат на карту памяти.

## Возможности

- измерение расстояния с помощью VL53L0X по I2C;
- запись данных на microSD-карту по SPI через FatFS;
- запуск сессии подсчёта по кнопке KEY;
- индикация состояния прохода синим LED;
- запись событий прохода;
- запись полного лога расстояний и состояний;
- сохранение текущего значения счётчика в отдельный файл;
- автономная работа от USB powerbank.

## Аппаратная часть

Используются следующие компоненты:

- STM32F401CCU6 Black Pill;
- ToF-датчик расстояния VL53L0X;
- модуль microSD-карты;
- microSD-карта;
- USB powerbank или другой источник питания через USB.

## Выходные файлы

Устройство записывает на microSD-карту три файла.

| Файл | Назначение |
|---|---|
| `COUNT.TXT` | Текущее значение счётчика проходов |
| `EVENTS.TXT` | События `BLOCKED` и `PASS` |
| `LOG.TXT` | Периодический лог расстояния, состояния и счётчика |

## Принцип работы

После подачи питания устройство переходит в режим ожидания. Подсчёт начинается после нажатия и отпускания кнопки KEY.

Датчик VL53L0X измеряет расстояние до противоположной стены. Если человек проходит между датчиком и стеной, измеренное расстояние становится меньше расстояния до стены. В этот момент система переходит в состояние `BLOCKED`.

Когда человек выходит из контролируемой зоны, расстояние возвращается к расстоянию до стены или датчик получает состояние `NO_TARGET`. В этот момент система возвращается в состояние `EMPTY`, увеличивает счётчик на единицу и записывает событие на карту памяти.

## Ограничения

Текущая версия использует один датчик расстояния. Поэтому прототип считает пересечения контролируемой зоны, но не определяет направление движения. Для определения направления можно расширить систему до двух датчиков или использовать другой принцип детектирования.
